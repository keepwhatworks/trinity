# Modules under src/trinity_local/ that are intentionally orphan from
# the production entry-point graph. Format: `path : reason`.
#
# Each line is a ratchet — when a new orphan appears (someone shipped
# code that's not reachable from main.py / mcp_server.py / the
# CORE_COMMAND_MODULES dispatch), CI fails until either:
#   1. The new module gets wired into a live caller, OR
#   2. It's added here with an explicit reason for why orphan is OK.
#
# Conversely, when an entry here stops being orphan (someone wired it
# up), CI also fails — remove the line so the whitelist reflects truth.
#
# See scripts/find_orphans.py for the analyzer.

src/trinity_local/retired_names.py : Registry consumed only by tests (test_retired_names_registry.py, test_doc_count_consistency.py). The point of the file IS to be a tests-only registry of dead names — production has nothing to learn from it at runtime.

src/trinity_local/me/correction_residual.py : The lens-as-seeker (basin_residuals / frontier) + axis_self_calibration. Measures the steer the four fixed taste axes can't name. Deliberately NOT wired into eval-prompt / council / dream per "prototype the meter, not the policy". Two 2026-06-02 retractions: (1) the first read "domain-local taste, no fifth axis" was wrong — axis_self_calibration shows the meter round-trips short text at residual 0.000; (2) my follow-up "register confound" was ALSO wrong (measured: short vs long prototypes give magnitude 0.945→0.954, coherence 0.105→0.106 — register adds ~nothing). The real flaw is the MEASURE + the grouping: residual_fraction divides by the content-dominated ‖pri−sac‖, and per-basin means reinforce topic (basin = topic cluster) — the one place content survives averaging. The VALIDATED path is content-invariant scoring: signed scalar shift c·a by sign-consistency recovers all 4 named axes at z=2.6–4.1; coherence-ranking (MRL of unit residuals) finds hidden structure where magnitude-ranking sees "all off-axis". basin_residuals/frontier (magnitude on per-basin means) are doubly confounded → stay orphaned. axis_self_calibration() is the in-register gate. Reachable via tests/test_correction_residual.py.

src/trinity_local/__main__.py : Package entry point — invoked as `python3 -m trinity_local [--mcp]`, never imported, so it has no in-tree callers by design (same as any __main__.py). Added v1.7.306 as a robust fallback tier for the Claude Code plugin's MCP launcher (plugins/trinity-local/bin/trinity-mcp): when the `trinity-local` console script isn't on PATH but the package is importable, the launcher runs `python3 -m trinity_local --mcp`. Re-exports main.main(). Reachable via the plugin launcher + `python -m`; guarded by tests/test_plugin_manifest.py (asserts the launcher names this invocation).


src/trinity_local/evals/baseline_floor.py : Tier-1 eval-trust gate / the reliable bug-finder (echo_gold POSITIVE control + echo_rejected/echo_prompt/empty NEGATIVE controls run through the real judge; refuses the headline when a dumb candidate isn't beaten or the judge can't recognize the user's own answer). It is what CAUGHT the broken-judge + reaction-as-prompt bugs neither founder nor agent predicted (2026-06-11). Needs the judge, so it lives at eval-RUN time; wiring into handle_eval_run as a pre-report gate is deferred to the eval unification (#316) to avoid a risky scorer change mid-refactor. Validated on real data (judge=claude); reachable via /tmp scratch proofs documented in memory eval_lens_unification_and_accretion_audit.
